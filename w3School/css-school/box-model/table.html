Block Level element

A block level element alway starts on a new line and takes up the full wifth available (streches out to the left and right as far as it can).

Examples of block level element:

  • < div >
  • < h1-h6 >
  • < p >
  • < form >
  • < header >
  • < footer >
  • < section >

Inline element

an inlne element does not start on a new line and only takes up as much width as neccessary

examples:

  • < span >
  • < a >
  • < img >

Using width, max-width and margin: auto;

As metioned in the previus chapter; a block-level element always takes up the full width available (streches out the eft and right as far as it can).

seting the width of a block element-level will revent it from streching out to the dges of its container. Then, you can set the margins to auto, to horinzontly center the element within its container. The element will take up the spcecified width, and the remaining space will be split equally between the two margins.

this < div > element has a width of 500px, and margin set to auto.

Nothe: the problem with this div above occurs when the borwser window is smaller than the width of the element. The browser then adds a horisontal scrollbar to the page.

using max-width instead, in this situation, will imporve the borwser's handling of small windows. This is important when making site usable on small devices:
