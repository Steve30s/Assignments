CSS Layout - The Position Property

The positon property specify the type of position method used for an element (static, relative, fixed, absolute or sticky).

There are five position values:

  • static
  • relative
  • fixed
  • absolute
  • sticky

Elements are then positoned using the top, bottom, left , and rigthproperties. However, these properties will not work unless the positon property is set first. They also work differently depending on the position value

Position : static;

HTML elements are by default

Static positioned elements are not afffected by the top, bottom, left, and right properties.

An element with positioned : static; is not positioned in any special way it is always positioned according to the normal flow of the page:

This div has a position element has a position : static;

Position : relative;

An element with position: relative; is positioned relative to its normal position. Other content will not be adjusted to fit into any gap left by the element.

This div has a position element has a position : static;

position: fixed;

An element wit position: fxed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are is used to position the element.

A fixed element does not leave a gap in the page where it would normally have been located.

this element has position fixed.

Position Absolute

An element with position: absolute; is positioned relative to the nearest positoned ancestor (instead of positioned relative the viewport, like fixed)

Hoever; if an absolute positioned element has no position ancestors, it uses the document body, moves along with the page scrolling

Note:A "positoned" element is one whose postion is anything except static

position: scticky;

An element with position: sticky; is positioned based on the user's scroll position.

A sticky element toggles between relative and fixed, depending on the scroll positon. It is posioned relative until a given offset position inthe viewport - then ot ' sticks' in place (like positon : fixed)

Css overflow

The overflow overflow property specify wheather to cip content or to add scrollbars when the content of an element is too big to fit in a specified area.

The overflow property has the following values:

  • visible - Default. The overflow is not clipped. It renders autoside the element's box
  • hidden - The overflow is clipped, and the rest of the content wil be invisible
  • scrool- The overflow is clipped, but the scrollbar is adde to see the rest of the content
  • auto- if overflow is clipped, a scrollbar should be added to see the rest of the content

The clear property

The clear property specidy what elementes can float besides the cleared element and on wich side.

The clear property can have one of the following values:

  • none - Allows floating elements on both sides. this is default
  • left - no floating elements allowed in the left side
  • right - No floating elements on the right side
  • both - No floating elements on either left or the right side
  • inherit - The element inhertis the cleat vallue of its parents

The most common way to use the clear property is after you have used a float on a elements

When clearing floats, you should match the clear to the left: if an element is floated to the left, then you should clear to the left. You floated element will continue to float, but the cleared element will appear bellow it on the webpage
